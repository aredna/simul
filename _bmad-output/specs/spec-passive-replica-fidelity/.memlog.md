- (direction) User approved autonomous implementation of a saved Replica fidelity policy and requires all current work and tests to be preserved.
- (decision) Passive Fidelity is the default policy for Isolated HTML; Conservative preserves the current stricter sanitizer; Strict Local Mirror remains hidden until no-network acceptance is genuinely complete.
- (capability) CAP-1: Users can persistently select Passive Fidelity or Conservative per companion settings; success is reload-stable policy routing with Passive Fidelity as the migration/default for Isolated HTML.
- (capability) CAP-2: Passive Fidelity preserves safely normalized visual CSS channels, queries, layers, pseudo-elements, fonts, HTTP(S) resources, local fragments, stylesheet order, media, disabled state, and cascade; success is deterministic fixtures matching those channels without executing website code.
- (capability) CAP-3: Passive Fidelity handles imports without silent loss by flattening readable loaded CSSOM and retaining sanitized HTTP(S) imports when unreadable; success is inline and linked import fixtures with diagnostics distinguishing flatten versus retained request.
- (capability) CAP-4: Passive Fidelity detects bounded ordinary CSSOM changes including insertRule, deleteRule, and mutable declarations; success is live replica updates without manual rebuild for deterministic CSSOM fixtures.
- (capability) CAP-5: Passive Fidelity preserves inert document semantics needed by styling and structure, including normalized HTTP(S) anchor href and responsive picture/source state, while preventing navigation, form submission, source mutation, playback, scripts, and active embedded documents.
- (capability) CAP-6: Passive Fidelity represents source blob visuals through Simul-owned temporary local resources or captured pixels when safely obtainable; success is no opaque source blob URL reuse and deterministic lifecycle revocation tests.
- (capability) CAP-7: Passive Fidelity preserves safely representable standards/quirks behavior, static media posters/current pixels, and a bounded computed-style fallback for otherwise unreadable presentation; success is bounded payload/performance tests and no indiscriminate computed-style serialization.
- (capability) CAP-8: Passive Fidelity preserves static inline and data-image SVG definitions, gradients, patterns, symbols, clips, masks, markers, filters, same-document references, and normalized passive HTTP(S) image, feImage, and external use resources where Chrome treats them as non-executable visuals.
- (capability) CAP-9: Content-free diagnostics report resource/style/SVG preserved, flattened, omitted, or blocked outcomes and reason categories, including whether replica-originated requests occurred or may occur.
- (capability) CAP-10: Deterministic fidelity and security fixtures cover cross-origin CSS/fonts, imports, CSSOM edits, backgrounds/fragments, SVG resources, inert anchors, responsive pictures, blob localization, dark/custom-element/nested-scroll cases, and continued blocking of scripts, handlers, JavaScript URLs, forms, and active embeds.
- (capability) CAP-11: Architecture, implementation spec, Settings disclosure/help, README, fidelity docs, version, and ready-to-load artifact reflect the shipped policies; success is a full green gate and fresh adversarial reviews.
- (constraint) Replica remains sandboxed, scriptless, pointer-inert, and unable to navigate, submit forms, mutate the source, activate media, instantiate website custom elements, or load active embedded documents.
- (constraint) No new permissions, sandbox weakening, remotely hosted executable code, or password/private-field transport is allowed without separate approval.
- (constraint) Passive Fidelity may make additional normalized HTTP(S) visual-resource requests from the replica and must disclose and diagnose that behavior.
- (constraint) Conservative behavior remains available as a saved fallback; existing privacy, translation, cache, OCR, scroll, and release tests remain intact.
- (constraint) Every CSS, SVG, blob, computed-style, and diagnostics path is bounded by bytes, rules, nodes, resources, time, and content-free logging.
- (constraint) No hostname-specific compatibility branches; OpenAI.com, Reddit, Y Combinator, and D-U-N-S are manual generic-compatibility checks only.
- (non-goal) Strict Local Mirror selection and a no-second-request guarantee are not shipped until the deferred pixel/resource substitution implementation and explicit no-network tests pass.
- (non-goal) Website JavaScript, custom-element constructors/lifecycle, interactive navigation/forms, frames/objects/embeds/portals/webviews, and video/audio playback are not enabled.
- (non-goal) SVG animation is not enabled in this pass; any future saved Allow passive visual animation option requires a separate narrow geometry/opacity/transform-only review and defaults off.
- (non-goal) Perfect reproduction of browser-inaccessible closed roots, protected cross-origin CSSOM, DRM/media pixels, or script-owned internal state is not claimed.
- (assumption) A saved global Replica fidelity preference is appropriate because the user requested one setting and did not request per-site policy overrides.
- (assumption) When source blob bytes cannot be read or captured through existing authority, Passive Fidelity omits the resource with a browser-inaccessibility diagnostic instead of requesting new permissions.

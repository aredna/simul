# Agent Guide

## Project intent

Simul is a small Chrome extension project used to learn the BMAD Method while
shipping quickly. Product behavior is intentionally undecided at bootstrap;
derive it from BMAD planning artifacts instead of inventing scope.

## Engineering baseline

- Use Node 24 LTS, npm, WXT, TypeScript, and Chrome Manifest V3.
- Keep production extension permissions empty until a documented requirement
  justifies each permission or host match.
- Keep browser entrypoints thin. Put testable, browser-independent logic in
  `lib/` and tests in `tests/`.
- Do not add remotely hosted executable code or commit secrets.
- Run `npm run check` before handing off a change.

## BMAD conventions

- `.agents/skills/` and `_bmad/` are generated by the official BMAD installer.
  Do not hand-edit generated files.
- Put durable team customizations in `_bmad/custom/config.toml`; personal
  customizations belong in ignored `*.user.toml` files.
- Store planning and implementation artifacts under `_bmad-output/` and
  long-lived project knowledge under `docs/`.
- For a small, well-understood feature, prefer `$bmad-quick-dev`. Use the full
  Product Brief -> PRD -> UX -> Architecture -> Epics/Stories path when the
  goal is learning the complete method or when scope is uncertain.
- Start each BMAD workflow in a fresh chat so workflow context does not bleed
  into the next phase.
